Week 6 Homework

In this exercise, you will build a small application that consumes "serverless" data from a public API of movies. This application will get movies "now playing" and allow the end-user to maintain a "watchlist" (i.e. watched vs. not watched) movies on the list.

The TMDB API

First, sign up for an account at https://themoviedb.org

Once verified and signed-in, go to Settings and create a new API key; in the form, indicate that you'll be using this API key for educational or personal use, and you should receive your new key right away.

For this exercise, we'll be using the "now playing" API endpoint https://api.themoviedb.org/3/movie/now_playing?api_key=${apiKey}&language=en-US

Note: image data returned by the API will only give you the filename; prepend with https://image.tmdb.org/t/p/w500/ to get the complete image URL.

Also note: the data from TMDB can change; it's live data with the latest movies "now playing". This is ok... keep calm and code on.

Requirements and Rubric

(These are high-level requirements for grading; more detailed mechanics are included in the comments of movies.js.)

  • Create a new brand-new repository in your GitHub account called hw6. Clone and open the code in VSCode. Add an HTML page and corresponding JS file – you can use the provided movies.html and movies.js as a starting point.
  • (2 points) Construct a URL to fetch playing now from TMDB. Write the contents of this array to the JavaScript console to ensure you've got good data.
  • (2 points) Loop through the Array of movies and insert HTML into the existing DOM element with the class name .movies to display the poster of each movie. Include a "watched" button to click for each movie.
  • (2 points) When the "watched button" is clicked, make the movie poster and corresponding "watched" button opaque. The result, at this point, should be a watchlist application that allows the user to click movies to mark them "watched", but will reset when the page is refreshed.
  • (4 points) Properly configure Firebase and Firebase Cloud Firestore in your application. Persist "watched" movies to Firestore; i.e. when you refresh the page, the lesser opacity of watched movies should persist.
